## ğŸ” API Endpoints

### Authentication
- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`

### Student (Authenticated User)
- `GET /api/v1/student/me`
- `PATCH /api/v1/student/me`

### Admin (Admin Only)
- `GET /api/v1/admin/students?page=1&limit=5`
- `PATCH /api/v1/admin/student-data/update/:studentId`
- `DELETE /api/v1/admin/student-data/delete/:studentId`

---

## ğŸ”„ Pagination

- Pagination implemented using query parameters

- Frontend controls page state

## ğŸ§  Key Design Decisions

- Enrollment date is system-generated (not editable)
- PATCH method used for partial updates
- Admin APIs protected using role-based middleware
- Students cannot access admin routes
- Redux Toolkit used for predictable state management
